# PDF Generation

The backend generates downloadable PDFs using WeasyPrint. HTML template construction is centralized in the `pdf_templates` service package.

## Architecture

- PDF engine: `weasyprint.HTML(string=html).write_pdf()`
- HTML template builders live in:
    - `backend/src/services/pdf_templates/invoice_template.py`
    - `backend/src/services/pdf_templates/purchase_template.py`
    - `backend/src/services/pdf_templates/ledger_template.py`
    - shared helpers: `backend/src/services/pdf_templates/builders.py`
- Route handlers call these template builders, then stream the generated bytes as `application/pdf`.

## Current PDF Endpoints

- `GET /api/invoices/{invoice_id}/pdf`
    - route: `backend/src/api/routes/invoices.py`
    - template builder: `_build_invoice_html` / `_build_multi_copy_invoice_html`

- `GET /api/ledgers/day-book/pdf`
    - route: `backend/src/api/routes/ledgers.py`
    - template builder: `_build_day_book_html`

- `GET /api/ledgers/{ledger_id}/statement/pdf`
    - route: `backend/src/api/routes/ledgers.py`
    - template builder: `_build_statement_html`

- `GET /api/payments/{payment_id}/pdf`
    - route: `backend/src/api/routes/payments.py`
    - currently uses route-local `_build_receipt_html`

## Request-to-PDF Flow

1. API route validates request and loads domain objects from DB.
2. Route calls a template builder to produce a full HTML document.
3. WeasyPrint renders HTML/CSS to bytes on A4 pages.
4. Route returns `StreamingResponse` with attachment filename.

## Why `pdf_templates` Exists

- Keeps route files focused on request/response and validation.
- Keeps HTML/CSS template code reusable and testable.
- Reduces duplication between sales, purchase, and ledger PDF rendering logic.

## Dependencies

Python package in `backend/requirements.txt`:

```txt
weasyprint==63.1
```

System libraries are required for text layout and font rendering.

macOS:

```bash
brew install pango
```

Debian/Ubuntu (already handled by backend Dockerfile):

```bash
apt-get install -y --no-install-recommends \
    libpango-1.0-0 \
    libpangocairo-1.0-0 \
    libpangoft2-1.0-0 \
    libgdk-pixbuf2.0-0 \
    libffi-dev
```

Alpine:

```bash
apk add --no-cache pango fontconfig ttf-freefont
```

## Troubleshooting

- `OSError: cannot load library 'libgobject-2.0-0'`:
    - Install system dependencies (`brew install pango` on macOS).
- Missing currency symbols or glyph fallback:
    - Install Unicode-capable fonts on host/container.
- Blank PDF output:
    - Validate generated HTML and inline CSS; run backend with debug logging.

## Conventions For New PDFs

- Put new HTML builders in `backend/src/services/pdf_templates/`.
- Re-export new builders from `backend/src/services/pdf_templates/__init__.py` when they are shared.
- Keep routes responsible only for orchestration, not template HTML/CSS.
